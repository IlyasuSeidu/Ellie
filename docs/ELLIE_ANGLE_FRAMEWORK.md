# Ellie Angle Framework

This is the canonical angle system for Ellie's stop-scroll content.

Machine-readable source:

- [ellie-angle-framework.js](/Users/Shared/Ellie/config/ellie-angle-framework.js)

Generator:

- [generate-stop-scroll-content.js](/Users/Shared/Ellie/scripts/generate-stop-scroll-content.js)

## Purpose

Use one stronger 20-angle canon across every day pack, then let each platform pull from its own ranked bank instead of reusing the same generic story shape everywhere.

## Master Canon

These are the 20 angles the generator can select from automatically.

| Angle                   | Lane          | What it does                                                   |
| ----------------------- | ------------- | -------------------------------------------------------------- |
| Stakes / Consequence    | Hook          | Leads with what goes wrong for the user if Ellie fails.        |
| User Empathy            | Human         | Turns product work into a real mining-life moment.             |
| Wrong Assumption        | Insight       | Reveals the mistaken belief behind the work.                   |
| Unexpected Challenge    | Hook          | Opens on friction, breakage, or the hard edge.                 |
| Technical Discovery     | Insight       | Teaches the technical lesson that mattered.                    |
| System Thinking         | Insight       | Shows how one product decision changes everything downstream.  |
| Design Tradeoff         | Insight       | Explains the hard interface or product choice.                 |
| Build Confession        | Human         | Admits the false start or thing that nearly shipped.           |
| Founder Opinion         | Insight       | States a sharp product point of view.                          |
| Before/After Reveal     | Proof         | Makes improvement instantly visible through contrast.          |
| Emotional Moment        | Human         | Centers the feeling that made the moment matter.               |
| Proof / Demo            | Proof         | Shows the thing working or the claim becoming visible.         |
| Mini Case Study         | Proof         | Packages the work into problem, move, and result.              |
| Myth-Bust               | Insight       | Knocks down the obvious but wrong explanation.                 |
| Identity / Worldview    | Human         | Ties the work to what Ellie stands for.                        |
| Search-Answer Explainer | Proof         | Answers the practical question people actually ask.            |
| Micro-Drama             | Hook          | Compresses the story into a small conflict with payoff.        |
| Audience Choice         | Participation | Lets people choose between directions or options.              |
| Open Loop / Cliffhanger | Participation | Ends with the unresolved edge that pulls to the next post.     |
| Community Debate        | Participation | Turns the day into a debate people can answer from experience. |

## Daily Selection Rule

The generator picks one angle from each lane:

1. Hook
2. Human
3. Insight
4. Proof
5. Participation

This avoids weak packs that are all explanation, all emotion, or all proof.

## Operating Rules

1. The strongest hook must still be true.
2. Human consequence beats internal process.
3. One day can blend multiple angles, but each asset should still feel native to its platform.
4. A high-performing pack usually combines one hook angle, one substance angle, and one participation angle.
5. If the repo evidence contradicts a dramatic line, the dramatic line loses.

## Ellie-Specific Bias

When in doubt, Ellie should bias toward:

1. Stakes / Consequence
2. User Empathy
3. Wrong Assumption
4. Proof / Demo
5. Founder Opinion

That bias matches the product: miners need a fast answer they can trust.
