# Hybrid Rewrite Style

This is the required second pass for every day pack.

## Goal

Keep the sharp tension, emotional framing, and audience-first storytelling of a strong creator draft while staying fully defensible against the actual repo evidence for that day.

## Core Rule

Be aggressive with tension.
Be conservative with facts.

## What To Keep

- High-stakes openings
- Clear human consequence
- Strong contrast lines
- Founder point of view
- Native platform structure
- Better hooks than the base generator

## What To Remove

- Inflated proof that the brief does not support
- Claims that collapse the day into a cleaner story than the repo actually shows
- Overdramatic language you cannot defend
- Builder-internal framing as the opening move
- Generic "follow for more" endings unless the platform truly needs one

## Rewrite Rules

1. Start with the user or the consequence, not the tooling.
2. Keep the strongest tension line if it is defensible.
3. If a dramatic line cannot be proven, rewrite it downward until it is true.
4. Translate technical work into human consequence.
5. Preserve chronology.
6. If day 1 included both invisible groundwork and the first visible layer, say both.
7. Replace internal proof with visible or human proof whenever possible.
8. Leave one sharp lesson and one strong audience question.

## Good Pattern

"Day 1 had two jobs: build the invisible foundation, then make sure the first visible layer sat on something truthful."

## Bad Pattern

"Day 1 had zero screens" when the repo clearly shows screens landed later that same day.

## Final Check

Before you keep a line, ask:

- Can I defend this from the day brief or the repo?
- Does this make the audience care faster?
- Is this the strongest true version of the sentence?
